# 用说话来控制家里设备——懒人福音

不想翻面板一个一个点？没问题。在聊天页面直接跟 AI 说「打开床头灯」「空调调到 26 度」「看看摄像头拍到什么」，AI 就会帮你操作。

就跟跟家里人说话一样自然。

---

## 一、背后发生了什么

你跟 AI 说一句话，它按这几步走：

```
你说话 → AI 先查设备列表 → AI 执行操作 → AI 确认真的生效了 → 回复你
```

每一步都有据可查，AI 不会凭空编造「已打开」——它真的去检查了设备状态才告诉你结果。每一步调用工具时，回复下面会冒出一张**工具卡片**（🔧 控制设备 / 📋 查询设备 / ✓ 确认操作），点开能看到传了什么参数、返回了什么，全程透明。

---

## 二、日常操作示例

### 开个灯

```
你：把床头灯打开
Aether：[📋 查询设备列表  ✓ 共 6 个设备]
        [🔧 开启 床头灯  ✓ 已生效，当前: on]
        [✓ 确认开启 床头灯  ✓ 已确认生效]
        已打开床头灯。
```

AI 自己会查设备、下发开灯命令、再确认灯真的亮了，一气呵成。你只管说话。

### 调温度

```
你：空调调到 26 度
```

AI 发现空调温度范围是 16-30°C，26 在范围内，直接调。如果用户说的是 35 度，超出了范围，AI 会调到最接近的合法值（30°C）。

### 切换模式

```
你：空调改成制热模式
```

AI 看到空调支持的模式有：关、制热、制冷、自动、除湿、送风。你说「制热」，它就在列表里找到对应项，执行切换。

如果你说的模式不在列表里，AI 会告诉你有哪几种可以选。

### 一条指令多个动作

```
你：客厅吊灯亮度调到 50%，风扇开中风
```

AI 会一个一个来，先把灯调了，再把风扇调了，互不干扰。

---

## 三、AI 必须遵守的规矩

这些规矩是写死在系统提示词里的，保证 AI 不会乱来：

- **必须先查设备**：不知道有什么设备就动手？不行。每次都要先 `get_entities` 看真实设备和可控项，`domain`/`service`/`param`/`entity_id` 都从返回里取，不自己拼造。
- **不能编造设备名**：你说「打开某某灯」，如果家里没有这个灯，AI 会老实告诉你不存在，不重试、不换 id 瞎试。
- **动了设备必须确认**：每次 `call_service` 操作完，都要 `verify_action` 检查是不是真的生效了。
- **参数不能超范围**：温度不能设到不存在的值，亮度不能调出范围。
- **多个操作逐个来**：不会把所有操作混在一起。
- **工具调用走 tool_call 机制**：不会在回复文本里写 JSON 代码块假装调工具。

---

## 四、条件控制——「如果……就……」

你还可以用条件来触发：

```
你：如果客厅温度超过 28 度就开风扇
```

AI 会按**三步走**：

1. 先调 `verify_condition` 检查「客厅温度超过 28 度」是不是真的（查设备实时状态）。
2. 如果是真的，就 `call_service` 执行开风扇。
3. 再 `verify_action` 确认风扇已经开了。
4. 如果温度没超过，就告诉你「温度没到，不用开」，**不执行**。

支持判断的条件类型：

- **设备状态**：比如「温度超过 28 度」「灯是不是开着」。
- **摄像头画面**：比如「摄像头里有人」。
- **天气**：比如「如果下雨」。
- **时间**：比如「晚上 8 点以后」。

三步（检查 → 执行 → 确认）缺一不可，AI 不会跳过任何一步。

## 五、摄像头相关的控制

### 根据画面触发

```
你：如果摄像头看到有人就开灯
```

AI 会先调 `describe_state` / `vision_chat` 拍一张画面、分析有没有人。如果有人就开灯，没人就告诉你没人。不会回「我无法判断」敷衍你。

### 直接问画面

```
你：摄像头现在看到什么？
```

AI 会拍一张当前画面，分析一下，然后把描述告诉你。比如「画面中客厅没有人，灯光开着，窗帘是拉上的」。

## 六、定时控制——「X 点 / 每天 / X 分钟后」

如果你说的是**未来某个时间点或周期**要做的事，AI 不会立即执行，而是建一个**定时任务**，到点自动跑：

```
你：每天早上 7 点开床头灯
Aether：[📅 创建定时任务 ...]
        好的，已创建定时任务：每天 7:00 开启床头灯。
```

AI 识别到「未来时间」语义后，会调 `scheduled_task_create` 工具，把执行内容和触发时间交给调度器，到点由系统自动执行——不占 LLM、不走聊天。

支持的时间说法：

- **一次性**：「明天早上 8 点关灯」「10 分钟后关空调」→ `at` 触发。
- **间隔周期**：「每小时刷新一次」「每 30 分钟关一次灯」→ `every` 触发。
- **cron 表达式**：复杂周期，AI 会自己翻译。

> 只有用户明确说「现在/马上」做时，AI 才会立即 `call_service`。详细看[定时任务管理](../04-自动化规则/定时任务管理.md)。

## 七、使用技巧

1. **直接用中文设备名**：你在 Home Assistant 里给设备起的中文名，直接用它叫就行，AI 按名字唯一匹配，不追问房间。
2. **说法不用太讲究**：「打开」「开一下」「开了」AI 都能听懂。
3. **一条消息多个操作**：可以一口气说「开灯、关窗帘、空调 25 度」，AI 逐个处理。
4. **想看 AI 在干嘛**：每条回复下面的工具卡片就是「透视眼」，点开能看到每一步调了什么工具、传了什么参数、结果如何，不用开任何开关。
5. **条件控制想清楚再说**：最好一次性把条件和结果都说清楚，比如「如果客厅温度超过 28 度就开风扇并关窗帘」。
6. **回答简短**：AI 调完工具会用自然语言简洁总结，不堆工具原文。

## 八、演示环境里可以控制的设备

Aether 自带一个 HA 模拟器，模拟了 6 个设备供你体验：

| 设备 | 房间 | 能干嘛 |
|------|------|--------|
| 床头灯 | 卧室 | 开关、调亮度 (0-100%) |
| 厨房灯 | 厨房 | 开关、调亮度 (0-100%) |
| 客厅吊灯 | 客厅 | 开关、调亮度 (0-100%) |
| 中央空调 | 客厅 | 开关、模式（关/制热/制冷/自动/除湿/送风）、温度 (16-30°C)、风速、风向 |
| 客厅窗帘 | 客厅 | 打开、关闭、停止、位置 (0-100%) |
| 客厅风扇 | 客厅 | 开关、风速（关/低/中/高）、摇头 |

这些设备通过 MQTT 接入 Home Assistant，AI 调用的是 HA 的标准服务接口。你在 HA 里加任何新设备，Aether 都能自动识别并控制，不用改代码。
