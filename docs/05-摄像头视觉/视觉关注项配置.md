# 视觉关注项配置

「视觉关注项」是告诉 Aether 的视觉模型：**看画面的时候，重点关心什么。**

默认情况下，模型关注的是「画面中的人和他们的行为」——也就是泛泛地描述谁在干什么。但这往往不够精准。比如你想让它专门盯着「猫有没有上沙发」「老人有没有跌倒」「餐桌上还有没有剩饭」，就需要设关注项。

## 一、关注项起什么作用

设了关注项之后，它会同时影响两个地方：

1. **摄像头面板识别**：模型对每帧画面做分类时，会带着你设的关注点去问，识别结果更聚焦。
2. **聊天系统提示**：启用的关注项文本会注入到 AI 管家的系统提示里，让它在对话里也知道「这户人家关心这些事」。

> 多条关注项之间是「或」的关系——只要画面符合其中任意一条，模型都会识别出来。它们不是互斥的。

## 二、在哪配

在侧边栏打开 **`/focus`（视觉关注项）** 页面。

页面很简单：

- 顶部一个输入框 + 「添加」按钮。
- 下方列表展示所有关注项，每条有「启用/关闭」开关和「删除」按钮。
- **没有编辑功能**：要改一条，删掉重新加。

## 三、增删查改

| 操作 | 入口 | 接口 |
|------|------|------|
| 查看全部 | 页面加载 | `GET /api/vision/focuses` |
| 新增 | 输入文本 → 添加 | `POST /api/vision/focuses`（body: `{ "text": "..." }`） |
| 启用/关闭 | 列表开关 | `PUT /api/vision/focuses/{id}`（body: `{ "enabled": true/false }`） |
| 删除 | 列表删除按钮 | `DELETE /api/vision/focuses/{id}` |

新增时默认就是启用状态。关闭（不删除）可以临时让某条不生效，又保留文本以后再开。

所有改动即时生效，并且持久化到数据库（`vision_focuses` 键），重启不丢。

## 四、不设会怎样

一条都不设时，模型用内置默认关注点：

> 画面中的人和他们的行为

这个默认值适合大多数「家里有人吗、在干嘛」的通用场景。如果你没有特殊盯防需求，可以完全不管这个页面。

## 五、写好关注项的小技巧

关注项本质上是一句自然语言，模型会尽量照着理解。写得具体、可观察，效果最好。

| 写法 | 效果 |
|------|------|
| ❌ 「注意安全」 | 太抽象，模型不知道看什么 |
| ✅ 「厨房灶台上是否有锅在加热」 | 具体、可观察，识别准 |
| ❌ 「猫」 | 模糊，是看猫在不在？还是看猫在干嘛？ |
| ✅ 「猫是否在沙发上」 | 明确的判断题 |
| ✅ 「玄关门口是否有人停留」 | 明确位置 + 行为 |

几个实用例子：

- 「客厅沙发上是否有人躺着」
- 「餐桌上是否有未收拾的餐具」
- 「老人是否长时间坐在同一位置未移动」
- 「阳台门是否开着」

## 六、和自动化规则的关系

关注项让模型「看得更准」，但它本身不触发任何动作。要让「看到某件事 → 自动做某件事」，需要去 `/task` 配**自动化规则**，把视觉条件写进规则里。

举个例子：
- 关注项：「猫是否在沙发上」（让模型识别更准）
- 自动化规则：条件「猫在沙发上」→ 动作「发消息提醒我」

两者配合，关注项提升识别质量，规则负责联动。规则的视觉触发机制详见《运动检测与视觉推理》和《视觉触发规则详解》。
