# 家庭信息与主题设置

打开 Aether 的设置页面（`/settings`），可以配置让系统「更像你的家」的东西：家庭名称、主人称呼、所在地，以及深色/浅色主题。

## 一、家庭信息

### 可以设置什么

| 设置项 | 字段 | 说明 |
|--------|------|------|
| 家庭名称 | `home_name` | 备用信息，目前主要影响天气上下文 |
| 主人称呼 | `owner_name` | 管家跟你打招呼时会用，比如「下午好，小童」 |
| 省份 | `province` | 天气定位 |
| 城市 | `city` | 天气定位 |
| 区县 | `district` | 天气定位 |

### 选位置：三级联动

Aether 用三级联动选择器选地点：

1. 先选**省份**
2. 选完省份，**城市**列表自动出现
3. 选完城市，**区县**跟着出来

每次改上级，下级自动重置，不会搞乱。直辖市（如上海）的「城市」就是省名本身。

### 主人称呼用在哪

`owner_name` 会在聊天界面的**问候浮层**里用到。打开聊天时，根据当前时间显示：

- 早上好 / 中午好 / 下午好 / 晚上好，`{主人称呼}`

比如设了「小童」，下午打开聊天会看到「下午好，小童」一闪而过（约 1.5 秒淡出）。没设主人称呼则只显示时间问候。

> 注意：主人称呼**只用于前端问候**，不会写进 AI 的系统提示词。管家的对话风格由人格（`/unique`）决定，和家庭信息无关。

### 保存后发生什么

点「保存设置」：

- 信息写入数据库（按用户存储）
- 触发 `home-info-changed` 事件，侧边栏主人名、天气组件立即刷新
- 按钮变绿显示「已保存」2 秒

### 天气查询地：以 `config.json` 的 `home` 段为准

家庭信息（主人名、家庭名等）按用户存储，但**天气查询地是全局的**——后端天气服务读 `config.json` 的 `home` 段，不跟用户走。这是家庭局域网场景的有意设计：一家人共用同一个地点的天气，没必要每人一份。

- 设置页改省市区 → 只更新 `user_settings.home_info`（侧边栏/家庭名用），天气不变。
- 想改天气查询地 → 改 `config.json` 的 `home` 段（`province`/`city`/`district`）然后重启。

## 二、深色 / 浅色主题

### 切换

设置页「系统偏好」区域有一个**深色模式**开关，点一下即时切换，不用重启。

- 开 = 深色（默认）
- 关 = 浅色

### 怎么记住的

主题存在浏览器本地（`localStorage` 键 `aether-theme`），下次打开还是你上次选的。在 App 启动第一帧就应用，不会闪一下白屏。

> 没有「跟随系统」自动模式，只能手动切深/浅。

### 界面小细节

**深色模式**：背景有流动渐变——几个彩色光斑缓慢漂移，20 秒一个循环。

**浅色模式**：同样的渐变但更淡，多了模拟「太阳光晕」的效果。

滚动条也专门设计过：细的、圆角的滑块，悬停时变色。

## 三、侧边栏

侧边栏固定在页面左边，是操作 Aether 的主要入口：

```
┌─────────────────┐
│  ✦ Aether       │  点这里回设备列表
│                 │
│  💬 管家        │  聊天 /chat
│  ⚙ 设置         │  系统设置 /settings
│  ⚡ 高级        │  高级设置 /advanced
│  📊 监控        │  监控 /monitor
│                 │
│  ┌───────────┐  │
│  │   14:32   │  │  实时时钟
│  │ 周一 6/22 │  │
│  └───────────┘  │
│  👤 小童        │  主人称呼 + 在线状态
│     ● 在线      │
└─────────────────┘
```

底部显示的是**主人称呼**（从 `/api/home/info` 读，没设则用登录用户名），不是家庭名称。旁边一个 `● 在线` 表示连接状态。

侧边栏还有点小动效：顶部 Logo 会「呼吸」发光，背景有微浮动光点。

### 手机上

用手机打开时，侧边栏自动缩窄到只显示图标，文字隐藏，不挤占屏幕。设置页布局也会自适应。

## 四、相关接口

| 接口 | 方法 | 作用 |
|------|------|------|
| `/api/home/info` | GET | 读取家庭信息 |
| `/api/home/info` | POST | 保存家庭信息（只更新非空字段） |

家庭信息按用户存储（`user_settings` 表，键 `home_info`），多用户体系下每个用户有自己的家庭信息。主题则是浏览器本地存储，每个浏览器各自记忆。
