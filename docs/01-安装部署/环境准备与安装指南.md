# 环境准备与安装指南

嘿，欢迎来到 Aether！这篇帮你把 Aether 智能家居助手在本机跑起来。跟着一步步来就行，不复杂。

## 你的电脑需要什么

| 项目 | 需要什么 |
|------|------|
| 操作系统 | Windows 10 或 11（64 位） |
| Python | 3.10 以上（用 conda 管理最省心） |
| Node.js | 18 以上（带 npm） |
| Docker | Docker Desktop（用来跑智能家居平台和消息中转） |
| 摄像头 | USB 摄像头或笔记本自带摄像头（没有也能用，只是看不了画面） |

## 先搞定 Python 环境

Aether 用 conda 管理 Python，环境名是 `yolo`。没建过的话，打开终端运行：

```powershell
conda create -n yolo python=3.10 -y
conda activate yolo
```

> 启动脚本会自动用 conda 的 `yolo` 环境跑 Python，平时不用手动激活。

## 拿到项目代码

```powershell
git clone <仓库地址> Aether-demo
cd Aether-demo
```

或者直接解压项目压缩包到你喜欢的地方。

## 安装 Python 依赖

激活 `yolo` 环境后，在项目根目录运行：

```powershell
conda activate yolo
pip install -r requirements.txt
```

等它装完就行，依赖列表项目里已经写好了。主要依赖：FastAPI（后端）、LangGraph（AI 编排）、OpenCV（视觉）、FAISS（向量检索）、croniter（定时任务）。

## 安装前端依赖

```powershell
cd frontend
npm install
cd ..
```

## 启动（Docker）

在项目根目录执行：

```powershell
docker compose up -d --build
```

首次构建约 5–10 分钟，之后启动很快。脚本会自动做这些事：

1. 构建 Aether 应用镜像（前端 vite build + 后端 pip install）
2. 启动 Docker 服务（MQTT 消息中转、Home Assistant 智能家居平台、Aether 应用）
3. 启动虚拟设备模拟器（模拟灯、空调、窗帘等，给 HA 喂数据）

启动完成后浏览器访问 `http://localhost:8010`。

想停止所有服务？

```powershell
docker compose down
```

## 各个端口干什么的

| 服务 | 端口 | 干嘛的 |
|------|------|------|
| 后端 API + 前端页面 | **8010** | Aether 的"大脑"，REST API + WebSocket，**生产/日常使用就访问这个端口**（后端直接托管前端构建产物） |
| 前端开发服务器 | **5173** | 仅改代码热重载时用（Vite 开发服务器，代理 /api /ws 到 8010）；生产环境不用 |
| 智能家居平台 | **8123** | Home Assistant 管理界面（Docker 容器） |
| 消息中转 | **1884** | Mosquitto MQTT，设备状态消息的"邮局" |

> 网页搜索用的是云端 **Exa MCP**，不占本地端口，详见「Exa 网页搜索配置」。

> 还有一个临时端口 **8011**：后端冷启动期间用它上报加载进度，让加载页能显示真实的启动阶段。启动完成后自动关闭。

## 怎么看日志

出问题了想看日志？日志都在 `logs/` 目录下：

```powershell
# 后端日志（最常用）
Get-Content logs\backend.log -Wait -Tail 20

# 虚拟设备模拟器日志
Get-Content logs\ha_simulator.log -Wait -Tail 20

# 前端日志
Get-Content logs\frontend.log -Wait -Tail 20

# MQTT 消息（Docker 容器日志）
docker logs mosquitto --tail 20 -f
```

后端日志带 `request_id`，每条请求都能顺着 ID 追踪完整链路，排查问题很方便（见「日志查看与问题排查」）。

## 手动启动（一般不这样用）

需要逐个启动服务来调试时，按这个顺序，每个服务新开一个终端：

```powershell
# 1. Docker 服务（MQTT + HA）
docker compose up -d

# 2. 虚拟设备模拟器
conda run -n yolo python ha_config\ha_simulator.py

# 3. Aether 后端（必须在项目根目录运行，或用 --app-dir 指定）
conda run -n yolo python -m uvicorn app.main:app --host localhost --port 8010 --app-dir "%CD%"

# 4. 前端开发服务器
cd frontend
npm run dev
```

## 生产部署：构建前端（访问 8010 而非 5173）

上面的一键启动和手动启动都走 **5173 开发服务器**，适合改代码时热重载。但日常使用或远程访问时，应该直接访问 **8010**——后端会托管前端构建产物，不用单独跑 Vite，外部设备也连得上（5173 只监听 `127.0.0.1`）。

要让 8010 能托管前端，**首次部署或前端代码更新后**，需要构建一次前端：

```powershell
cd frontend
npm run build
```

这一步会做两件事：

1. `vite build` 把 Vue 源码编译成静态文件（产出 `frontend/dist/`）
2. `node scripts/sync-to-backend.js` 自动把 `dist/` 复制到 `app/static/frontend/`，后端启动时从这里托管

构建完成后，只要后端在 8010 跑着，浏览器访问 `http://localhost:8010` 就能看到完整界面。

> **什么时候要重新构建？** 只要改了 `frontend/src/` 下的前端代码（.vue / .js），想让 8010 上的页面跟着更新，就重新跑一次 `npm run build`。只改后端代码不用重建。
>
> **Docker 部署**不用手动构建——`docker compose up -d --build` 会在镜像里自动完成前端构建，详见「Docker 服务部署指南」。

## 第一次进来会怎样

打开浏览器后会经过几步：

1. **欢迎页**（`/landing`）：一段视频，点"点击进入"。
2. **加载页**（`/loading`）：检测后端是否就绪。冷启动时后端要加载模型、连摄像头，可能等十几秒。
3. **登录页**（`/login`）：第一次需要注册一个账号（用户名 + 密码 + 可选显示名）。注册即登录。
4. **设置向导**（`/setup`）：如果还没配置过，会引导你填家庭信息和 LLM 密钥（2 步）。
5. **聊天页**（`/chat`）：正式开聊。

之后每次进来刷新，都会先过加载页确认后端就绪，再进主界面。

搞定了！环境搭好之后，接下来就可以配置各种服务连接了——Home Assistant、LLM 密钥、天气等等，每篇都有专门文档。
