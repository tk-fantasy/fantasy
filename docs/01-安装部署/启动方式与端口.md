# 启动方式与端口

Aether 支持两种启动方式，端口互不冲突，**可同时运行**。

## 端口分配

| 端口 | bat 本机模式 | Docker 容器模式 | 说明 |
|------|-------------|----------------|------|
| 9010 | ✅ 主服务 | — | bat 模式 API + WebSocket + 前端 SPA |
| 9011 | ✅ 进度端口 | — | bat 模式冷启动进度（加载页轮询） |
| 8010 | — | ✅ 主服务 | Docker 模式 API + WebSocket + 前端 SPA |
| 8011 | — | ✅ 进度端口 | Docker 模式冷启动进度 |
| 5173 | ✅ dev 模式 | — | vite dev server（仅 `run_demo_fixed.bat`） |
| 1884 | ✅ 共用 | ✅ 共用 | MQTT 消息中转（侧车容器） |
| 8123 | ✅ 共用 | ✅ 共用 | Home Assistant 设备模拟（侧车容器） |

> **设计原因**：bat 和 Docker 各用独立端口（9010 vs 8010），因此可以同时运行，互不影响。MQTT 和 HA 是共用的侧车容器，两种模式都连接同一组容器。

---

## 方式一：bat 本机启动（日常使用 / 开发）

### 日常使用（生产模式 + PWA）

双击桌面快捷方式 **Aether**（或直接双击 `run_app.bat`）：

1. 自动启动 Docker 侧车容器（MQTT + HA）
2. 构建前端（`vite build`，含 PWA 产物）
3. 启动后端 uvicorn（端口 9010）
4. 自动打开浏览器 `http://localhost:9010`

浏览器地址栏右侧会出现**「安装」图标**，点击安装后：
- 桌面生成 Aether 图标
- 打开后是独立窗口（无地址栏），体验类似原生 app

### 开发模式（热重载）

双击桌面快捷方式 **Aether (开发)**（或直接双击 `run_demo_fixed.bat`）：

1. 自动启动 Docker 侧车容器
2. 启动后端 uvicorn（端口 9010）
3. 启动 vite dev server（端口 5173，热重载）
4. 自动打开浏览器 `http://localhost:5173`

> dev 模式不启用 PWA service worker，改前端代码后浏览器自动刷新。

---

## 方式二：Docker 容器启动（部署）

```bash
docker compose up -d
```

一次性启动全部三个服务（mqtt + homeassistant + aether），Aether 应用在容器内运行（端口 8010），前端构建产物已打进镜像。

访问 `http://localhost:8010`，同样支持 PWA 安装。

### 前提条件

1. `cp config.example.json config.json`，把 `ha.token` 改成 HA 长期令牌
2. `cp .env.example .env`，填入 LLM / STT 密钥
3. 已构建 `aether-ha:local` 镜像（HA 配置见 `ha_config/`）
4. 已构建 aether 镜像（`docker compose build aether`）

---

## 同时运行

两种模式端口不冲突，可以同时启动：

- bat 模式 → `http://localhost:9010`
- Docker 模式 → `http://localhost:8010`

> 注意：两者共用同一个 SQLite 数据库（`app/data/aether.db`），并发写入时可能有短暂锁竞争，但不影响正常使用。

---

## 桌面快捷方式

| 快捷方式 | 指向 | 模式 | 端口 |
|----------|------|------|------|
| Aether | `run_app.bat` | 生产 + PWA | 9010 |
| Aether (开发) | `run_demo_fixed.bat` | 开发热重载 | 5173 (前端) / 9010 (后端) |

## Emoji 索引重建

`emoji_index.json`（~119MB，5225 个 emoji 的语义向量索引）未纳入版本控制，换机器克隆后需重建：

1. 确保已在「密钥管理」页配置 embed 模型（如 `BAAI/bge-m3`）
2. 进入「设置」页 → 「Emoji 索引」区块
3. 点击「重建索引」按钮
4. 等待进度条完成（5225 个 emoji 逐个调用 embed API，约需几分钟）

重建过程后台异步执行，可关闭页面，通过重新进入设置页查看进度。

详见 [Emoji 图标自定义](../07-个性化/Emoji图标自定义.md)。
