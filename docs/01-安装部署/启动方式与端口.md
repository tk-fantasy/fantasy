# 启动方式与端口

## 启动

```bash
docker compose up -d --build
```

首次构建约 5–10 分钟（前端 npm ci + vite build、后端 pip install）。构建完后等待约 20 秒后端启动完成，浏览器访问 `http://localhost:8010`。

停止：

```bash
docker compose down
```

## 端口

| 端口 | 服务 | 说明 |
|------|------|------|
| 8010 | Aether | API + WebSocket + 前端 SPA |
| 8011 | Aether 进度 | 冷启动进度（加载页轮询） |
| 1884 | MQTT | 消息中转 |
| 8123 | Home Assistant | 设备模拟 |

## 首次启动前提

1. Docker Desktop 已启动
2. `cp config.example.json config.json`，把 `ha.token` 改成 HA 长期令牌
3. `cp .env.example .env`，填入 LLM / STT 密钥

（HA 用官方镜像 `ghcr.io/home-assistant/home-assistant:stable`，配置通过 `ha_config/` 挂载，`docker compose up` 自动拉取，无需手动构建）

## 国内镜像加速

基础镜像（node、python）默认走 Docker Hub，国内拉取极慢。已配置 registry-mirror（见 `~/.docker/daemon.json`），改 mirror 后需重启 Docker Desktop 生效：

```json
{
  "registry-mirrors": ["https://docker.m.daocloud.io"]
}
```

npm / pip 的国内源已在 Dockerfile 内写死（npmmirror / 清华），无需额外配置。

## Emoji 索引重建

`emoji_index.json`（~119MB）未纳入版本控制，换机器后需重建：

1. 在「密钥管理」页配置 embed 模型（如 `BAAI/bge-m3`）
2. 进入「设置」页 → 「Emoji 索引」→ 点击「重建索引」

详见 [Emoji 图标自定义](../07-个性化/Emoji图标自定义.md)。
