# 启动方式与端口

## 启动

双击桌面 **Aether** 快捷方式（或双击 `run_app.bat`）：

1. `docker compose up -d --build` 启动全部服务（MQTT + HA + Aether）
2. 等待后端就绪
3. 自动以 Edge app 模式打开 `http://localhost:8010`（独立窗口，无地址栏）

手动启动：

```bash
docker compose up -d --build
# 等待约 20 秒后端启动完成
msedge --app=http://localhost:8010 --start-maximized
```

停止：

```bash
docker compose down
```

## 端口

| 端口 | 服务 | 说明 |
|------|------|------|
| 8010 | Aether | API + WebSocket + 前端 SPA |
| 8011 | Aether 进度 | 冷启动进度（加载页轮询） |
| 1884 | MQTT | 消息中转 |
| 8123 | Home Assistant | 设备模拟 |

## App 窗口原理

`msedge --app=URL` 是 Chromium 内置的 app 模式：

- 用 Edge 内核渲染网页
- 去掉地址栏、标签页栏、书签栏
- 只保留网页内容 + 窗口标题栏
- `--start-maximized` 启动时最大化

跟普通浏览器标签页的区别就是没有浏览器 UI，看起来像原生 app。

## 首次启动前提

1. Docker Desktop 已启动
2. `cp config.example.json config.json`，把 `ha.token` 改成 HA 长期令牌
3. `cp .env.example .env`，填入 LLM / STT 密钥
4. 已构建 `aether-ha:local` 镜像（HA 配置见 `ha_config/`）

## Emoji 索引重建

`emoji_index.json`（~119MB）未纳入版本控制，换机器后需重建：

1. 在「密钥管理」页配置 embed 模型（如 `BAAI/bge-m3`）
2. 进入「设置」页 → 「Emoji 索引」→ 点击「重建索引」

详见 [Emoji 图标自定义](../07-个性化/Emoji图标自定义.md)。
