# Home Assistant 连接配置

这篇帮你把 Aether 和智能家居平台（Home Assistant，简称 HA）连起来。连上之后，Aether 就能看到你的设备、控制你的灯和空调了。

## 两个容易搞混的页面

先说清楚，Aether 有两个跟 HA 有关的页面，别走错：

| 页面 | 路径 | 干什么 |
|------|------|------|
| **HA 连接配置** | `/ha` | 配 HA 的地址和访问令牌（这页） |
| **设备面板** | `/halist` | 浏览和控制所有设备（另一篇文档讲） |

这篇只讲 `/ha`——怎么把连接配通。设备怎么控制在「设备面板操作指南」里。

## 怎么到 `/ha` 页

侧边栏没有 HA 的直接入口，有三种方式到：

1. 在聊天框输入 `/ha` 斜杠命令，直接跳过去
2. 首次设置向导结束后如果设备没连上，也会提示你来这里
3. 浏览器直接访问 `http://localhost:5173/ha`

## 开始之前

确保你已经：

- Docker 里的 Home Assistant 容器在运行（`docker compose ps` 看到 `aether-ha` 是 `Up`）
- 在 HA 里创建了管理员账号（第一次进 `http://localhost:8123` 时创建）
- 拿到了长期访问令牌（见下方）

## 拿长期访问令牌

1. 浏览器打开 `http://localhost:8123`，登录 HA
2. 点左下角你的头像 → 进入个人资料页
3. 拉到最下面找到「长期访问令牌」（Long-Lived Access Tokens）
4. 点「创建令牌」，起个名字（比如 `aether`），复制那一长串字符

> 这个令牌只显示一次，复制好存起来。丢了就得重新创建。

## 在 `/ha` 页配置连接

页面分三块：

### 连接状态

显示当前的 HA 地址和 Token（打码预览），带"已配置/未配置"徽标。

### 连接配置

- **URL**：填 HA 的地址，本地默认 `http://localhost:8123`
- **Token**：粘贴刚才拿到的长期访问令牌。留空表示不修改（已配置过的话）
- 点 **「保存配置」**

如果你把 HA 装在别的电脑上，把 `localhost` 换成那台电脑的 IP（同局域网）或 Tailscale 地址（远程，见「Tailscale 远程访问」）。

### 连接测试

点 **「测试」** 按钮（URL 为空时按钮禁用）：

- **连接成功**：绿色提示，告诉你找到了几个设备
- **连接失败**：红色错误，检查地址、Token、HA 是否在运行

## Aether 是怎么跟 HA 沟通的

配通后 Aether 会自动做这些：

- **看设备状态**：定时拉取所有设备的当前状态，缓存 60 秒（后台每 60 秒刷新一次设备目录）
- **控制设备**：通过 `call_service` 调用 HA 的服务（开灯、调温度等）
- **了解能力**：拉取 HA 支持哪些服务、每个服务需要什么参数，动态推导设备控件
- **校验动作**：控制后用 `verify_action` 只读查证设备是否真的变了状态

设备目录会缓存成文本喂给 AI，所以你聊天说"把客厅的灯打开"，AI 知道有哪些灯、在哪个区域、叫什么名字。

## 虚拟设备模拟器

Aether 自带虚拟设备模拟器 `ha_config/ha_simulator.py`，通过 MQTT 往 HA 报告虚拟设备状态。`docker compose up -d` 会自动启动它。

| 设备 | 在哪个房间 | 能干嘛 |
|------|-----------|--------|
| 床头灯 | 卧室 | 开关 + 调亮度 |
| 厨房灯 | 厨房 | 开关 + 调亮度 |
| 客厅吊灯 | 客厅 | 开关 + 调亮度 |
| 中央空调 | 客厅 | 模式、温度、风速、摆风 |
| 客厅窗帘 | 客厅 | 打开/关闭（0~100） |
| 客厅风扇 | 客厅 | 开关、风速、摇头 |

模拟器启动后会把设备状态发到 MQTT，HA 从 MQTT 收到，Aether 再从 HA 拿到。链路是：模拟器 → MQTT → HA → Aether。

## 出问题了怎么办

### 地址填错了

**现象**：测试提示 `连接失败: 连接被拒绝`

**解决**：
- 确认 HA 容器在运行：`docker compose ps`（看 `aether-ha` 是不是 `Up`）
- 确认地址用 `http://`（不是 `https://`），端口 `8123`
- HA 在别的电脑上时，确认防火墙放行了 8123

### Token 不对

**现象**：提示 `HTTP 401: Unauthorized`

**解决**：
- 检查 Token 有没有完整复制（前后别多空格换行）
- 去 HA 重新创建一个令牌
- 确认 Token 没被撤销（HA 个人资料页可以管理）

### HA 没启动

**现象**：Aether 启动后设备列表空的

```powershell
docker compose ps
docker compose logs homeassistant --tail 30
docker compose up -d homeassistant
```

### 模拟器没连上 MQTT

**现象**：HA 能打开但没设备，或设备状态不更新

- 确认 MQTT 容器在运行：`docker compose ps`（看 `mosquitto`）
- 确认模拟器在跑，看日志：`Get-Content logs\ha_simulator.log -Tail 20`
- 确认 1884 端口没被占（Aether 用 1884，不是默认 1883）

### 设备没显示房间

**现象**：设备面板里「区域」是空的

**解决**：在 HA 里把设备分配到区域（设置 → 设备与服务 → 实体 → 归属区域）。

搞定了！连上 HA 之后，Aether 就能帮你管设备了。接下来可以配置天气、LLM 密钥等。
