# AI 人格自定义指南

嫌 AI 管家说话太正经？想让它像个老朋友，或者像个严谨的工程师？你可以自己定义它的**人格**。这篇告诉你怎么调。

---

## 1. 能调什么，不能调什么

Aether 的系统提示词由两部分组成：

| 部分 | 能不能改 | 在哪改 |
|------|----------|--------|
| **角色设定（persona）** | ✅ 能改 | `/unique` 页 |
| **行为原则（guidelines）** | ❌ 系统管 | 代码里写死 |

- **persona** 决定「它是谁」——身份、性格、说话风格、边界。比如「你是 Aether，简洁务实，懂技术又不啰嗦」。
- **guidelines** 是一套固定的行为规矩——比如控制设备前要先查、动了要核对、不知道就说不知道、定时任务要建调度不立即执行。这部分**不开放修改**，保证 AI 行为稳定可控。

所以你能调的就是 persona 这一个文本框。别小看它，写好了 AI 的语气能完全变个样。

---

## 2. 默认人格长什么样

如果你没自定义过，用的是默认 persona：

> 你是 Aether，一个本地家庭智能助手，运行在用户自己的电脑上，连着一个摄像头，通过 Home Assistant 控制全屋智能设备。你的性格：简洁、务实，懂技术又不啰嗦。说话用中文，口语化，一次把事说清楚，不堆废话。

配套的系统原则（不可改）大致包括：

- **能力边界**：只能控 HA 设备、看摄像头、搜网页、验证状态。做不到的事直接说「我做不到」。
- **工具用法**：动作前先 `get_entities` 看真实设备和可控项；`verify_condition`/`verify_action` 只读；条件指令三步走（验条件 → 执行 → 核对）。
- **定时任务**：用户指定未来时间做事时，必须建调度任务，不立即执行。
- **诚实**：以工具返回为准，没调工具就说不知道，不描述没发生的事。

---

## 3. 怎么改人格

1. 在聊天框打 `/unique`，或侧边栏进「助手人格」页。
2. 看到「🎭 角色设定」这一栏，下面一个大文本框。
3. 如果之前自定义过，标题旁边会有个「已自定义」小标签。
4. 在文本框里写你想让 AI 扮演的角色设定，写完点「保存设置」。
5. 按钮会变成「保存中...」→「已保存」，2 秒后恢复。**新的对话立刻生效**，不用重启。

调的是 `chat_assistant.persona`，存在 `config.json` 里，跨会话持久。

---

## 4. 写个好 persona 的几个建议

persona 是给 LLM 的角色指令，写得清楚它才听得住：

- **说清身份**：「你是 XX，一个……」开头，告诉它自己是谁。
- **定性格和语气**：比如「说话像邻居大爷，热乎但不啰嗦」「像个严谨的工程师，先说结论再补细节」。
- **给边界**：哪些话题聊、哪些不聊，比如「不聊政治」「不主动推荐商品」。
- **别写具体能力**：能力清单是系统根据真实工具动态拼的，你写了反而可能对不上。只写「性格、语气、边界」就好。
- **别太长**：几句话到一段话就够了，太长会挤占上下文。

### 几个例子

**温柔管家型：**
```
你是小艾，家里的贴心管家。说话温柔、有耐心，像个操心的大姐。每次回答先安抚再办事，遇到做不到的会解释原因并给替代建议。称呼用户为「主人」。
```

**极简工程师型：**
```
你是 Aether，一个沉默寡言的工程师。回答尽量短，能用一句话说清就不用两句。不寒暄、不废话，直接给结论和操作结果。技术细节用户问起再说。
```

**育儿助手型：**
```
你是兜兜，一个懂育儿的家庭助手。回答问题时如果有涉及孩子的安全、健康，会多提醒一句注意事项。语气轻松，偶尔用个 emoji，但不啰嗦。
```

---

## 5. 恢复默认人格

想改回默认？把文本框**清空**，然后点「保存设置」就行——后端检测到 persona 为空，会自动用回 `DEFAULT_PERSONA`，「已自定义」标签也会消失。

---

## 6. 系统提示词到底拼了什么

你写的 persona 只是系统提示词的**开头**。完整的系统提示词按这个顺序拼起来（这部分你不用管，了解即可）：

1. **persona**（你写的，或默认的）
2. **guidelines**（系统原则，固定）
3. **当前时间**（自动注入，防止 AI 编时间）
4. **能力清单 + 设备目录**（根据真实工具和 HA 设备动态生成）
5. **设备可控项**（当前查询匹配到的设备，列出 service/param）
6. **天气快照**（有配置天气 API 时）
7. **历史摘要**（对话压缩后生成的摘要）

所以你只管把 persona 写好，能力和设备信息系统会自己补，不会对不上。
