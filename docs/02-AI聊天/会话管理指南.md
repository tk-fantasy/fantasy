# 会话管理指南

聊得多了，会话也会多起来。Aether 把每次对话都存成一个个**会话**，你可以在「会话管理」页浏览、切换、删除它们。这篇告诉你怎么管。

---

## 1. 会话是什么

每当你开始一段新对话（比如打个 `/new`，或者第一次进聊天页自动建一个），后端就创建一个会话：

- 每个会话有自己的 **id**（一串 UUID）、**消息条数**、**创建时间**、**最后更新时间**。
- 会话是**按用户隔离**的——多用户环境下，你只能看到自己的会话，别人看不到。
- 会话存在后端，关浏览器、换设备都不丢，登录后还能接着聊。

---

## 2. 打开会话管理页

两种方式：

- 侧边栏点「会话管理」，或在聊天框打 `/sessions`。
- 路由是 `/sessions`。

页面很简单：一个会话卡片列表，每张卡片显示：

```
a1b2c3d4-...                12 条消息
创建  2026-07-07 14:30    更新  2026-07-07 15:05                                    ✕
```

- 左边是会话 id 和消息数徽章。
- 中间是创建时间和最后更新时间。
- 右边一个 ✕ 删除按钮。

---

## 3. 切换会话

点一下任意一张卡片，就跳到聊天页，并自动加载那个会话的历史消息：

- 路由变成 `/chat?session=<会话id>`。
- 之前聊的内容全部还原，可以接着往下聊。
- 当前会话 id 会记在浏览器的 `sessionStorage` 里，刷新页面不会丢。

> 想新建一个会话？在聊天页打个 `/new` 就行，旧会话不受影响，随时能切回来。

---

## 4. 删除会话

点卡片右边的 ✕ 按钮，会立即删除该会话（目前没有二次确认弹窗，点之前想清楚）：

- 调用 `DELETE /api/sessions/{id}`。
- 删除后卡片从列表消失，会话里的消息也没了，**不可恢复**。
- 删除按钮在删除过程中会变成 ⏳ 等待图标。

---

## 5. 没有重命名

目前会话**不支持重命名**——列表里显示的就是会话 id（UUID），靠时间和消息数来区分。如果你找某段对话，按更新时间从新到旧找通常最快。

---

## 6. 在聊天页对会话做的操作

会话管理页只管「切换」和「删除」。其他对当前会话的操作都在聊天页里用斜杠命令完成：

| 命令 | 作用 | 说明 |
|------|------|------|
| `/new` | 新建会话 | 开个空白会话，旧的保留 |
| `/undo` | 撤销上一轮 | 抹掉最近一轮（你一句 + AI 一句），相当于「刚才那轮不算」 |
| `/clear` | 清空消息 | 把当前会话消息全清空，会话本身还在（有确认弹窗） |
| `/compress` | 压缩摘要 | 把历史对话压成摘要，省 token 但保留重点 |

这几个命令的细节看[斜杠命令参考](./斜杠命令参考.md)。

---

## 7. 后端 API 一览

会话相关的接口都在 `/api/sessions` 下，前端用到的：

| 方法 | 路径 | 作用 |
|------|------|------|
| POST | `/api/sessions` | 创建新会话 |
| GET | `/api/sessions` | 列出当前用户所有会话摘要 |
| GET | `/api/sessions/{id}` | 获取单个会话详情（含全部消息） |
| DELETE | `/api/sessions/{id}` | 删除会话 |
| POST | `/api/sessions/{id}/undo` | 撤销最后一轮 |
| POST | `/api/sessions/{id}/clear` | 清空消息 |
| POST | `/api/sessions/{id}/compress` | 生成摘要 |

> 还有一个 `POST /api/sessions/{id}/fork`（从某个消息处分叉出新会话）后端已实现，但前端目前没有暴露入口，留着以后用。
