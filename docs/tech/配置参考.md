# 配置参考

本文档是 `config.json` 和 `.env` 的完整配置 schema 参考，以 `config.example.json` 为准。

---

## 1. config.json 完整 Schema

### 1.1 llm_keys — LLM 密钥定义

数组，每个元素定义一组 API 密钥。

```json
{
  "id": "api-c12091",
  "base_url": "https://open.bigmodel.cn/api/paas/v4",
  "model": "glm-4-flash",
  "type": "chat",
  "chat_path": "/chat/completions",
  "embed_path": "",
  "api_key_env": "ZHIPU_API_KEY_2"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | 是 | 密钥唯一标识，供 providers 引用 |
| `base_url` | string | 是 | API 基础地址（不含路径） |
| `model` | string | 是 | 模型名称 |
| `type` | string | 是 | `chat` / `vision` / `embed` / `summary` / `stt`（5 种角色） |
| `chat_path` | string | 否 | Chat 接口路径，默认 `/chat/completions` |
| `embed_path` | string | 否 | Embeddings 接口路径，默认 `/embeddings` |
| `api_key_env` | string | 是 | `.env` 中对应的环境变量名 |

> 实际部署中 `llm_keys` 存在数据库 `user_settings` 表（按用户），`config.json` 里的是启动时加载的第一个用户的快照。

### 1.2 providers — 角色→密钥映射

```json
{
  "chat":     { "key_id": "api-c12091", "max_concurrency": 8, "enabled": true, "thinking": false },
  "summary":  { "key_id": "open-f2ef5b", "max_concurrency": 8, "enabled": true, "thinking": false },
  "vision":   { "key_id": "open-fcc2a5", "max_concurrency": 8, "enabled": true, "thinking": false, "multimodal": true },
  "embed":    { "key_id": "api-c12091", "max_concurrency": 8, "enabled": true },
  "stt":      { "key_id": "open-f2ef5b", "max_concurrency": 4, "enabled": true }
}
```

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `key_id` | string | - | 指向 `llm_keys[].id` |
| `max_concurrency` | int | 8 | 该角色最大并发请求数 |
| `enabled` | bool | true | 是否启用 |
| `thinking` | bool | false | 是否启用推理链（深度思考） |
| `multimodal` | bool | true | vision 专用：是否支持多模态图像 |

5 种角色：`chat`（对话）/ `vision`（视觉）/ `summary`（摘要）/ `embed`（嵌入、Emoji 搜索、RAG）/ `stt`（语音转文字）。

### 1.3 vision — 视觉子系统

```json
{
  "downscale_max_side": 448,
  "jpeg_quality": 70,
  "motion_hash_size": 16,
  "motion_threshold": 15,
  "motion_check_interval_seconds": 0.2,
  "min_infer_interval_seconds": 3.0,
  "max_idle_interval_seconds": 60.0,
  "vision_use_img_count": 3,
  "frame_interval_ms": 1000,
  "camera_index": 0,
  "read_retry_count": 3,
  "read_retry_interval_seconds": 0.1,
  "release_cooldown_seconds": 0.8,
  "max_backoff_seconds": 15.0
}
```

| 字段 | 默认 | 说明 |
|------|------|------|
| `downscale_max_side` | 448 | VL 输入图像最大边长 (px) |
| `jpeg_quality` | 70 | MJPEG/base64 编码质量 |
| `motion_hash_size` | 16 | dHash 尺寸（16→256 位） |
| `motion_threshold` | 15 | 运动触发阈值（Hamming 距离） |
| `motion_check_interval_seconds` | 0.2 | 运动检测间隔 |
| `min_infer_interval_seconds` | 3.0 | 两次推理最小间隔 |
| `max_idle_interval_seconds` | 60.0 | 无运动时最大推理间隔（心跳兜底） |
| `vision_use_img_count` | 3 | 单次推理帧数 |
| `frame_interval_ms` | 1000 | 抓帧间隔 |
| `camera_index` | 0 | 摄像头设备号 |
| `read_retry_count` | 3 | 读帧失败重试次数 |
| `read_retry_interval_seconds` | 0.1 | 重试间隔 |
| `release_cooldown_seconds` | 0.8 | 摄像头释放冷却 |
| `max_backoff_seconds` | 15.0 | 设备掉线重连退避上限 |

### 1.5 chat_assistant — 对话助手设定

```json
{
  "persona": "你是Aether，一个本地家庭智能助手...",
  "capabilities": "你能做这些事...",
  "guidelines": "几条原则..."
}
```

| 字段 | 说明 |
|------|------|
| `persona` | 系统人格，可通过 `/api/unique` 修改 |
| `capabilities` | 能力列表（前端只读，空则用内置默认） |
| `guidelines` | 行为准则（前端只读，空则用内置默认） |

> 只有 `persona` 用户可改；`capabilities`/`guidelines` 由系统管理。系统提示组装顺序：persona → guidelines → 时间 → 设备目录 → 控件 → 天气 → 摘要。

### 1.6 rag — 对话检索增强

```json
{
  "recent_turns": 5,
  "retrieve_top_k": 6,
  "retrieve_top_n": 3,
  "soft_max_turns": 12,
  "hard_max_turns": 16,
  "soft_max_tokens": 12000,
  "hard_max_tokens": 16000,
  "soft_max_chars": 24000,
  "hard_max_chars": 32000,
  "summary_blocks": 2
}
```

| 字段 | 默认 | 说明 |
|------|------|------|
| `recent_turns` | 5 | 保留最近 N 轮 |
| `retrieve_top_k` | 6 | 检索候选数 |
| `retrieve_top_n` | 3 | 注入 prompt 的轮数 |
| `soft_max_turns` | 12 | 软限制，触发摘要 |
| `hard_max_turns` | 16 | 硬限制，强制截断 |
| `soft_max_tokens` / `hard_max_tokens` | 12000 / 16000 | token 软/硬限制 |
| `soft_max_chars` / `hard_max_chars` | 24000 / 32000 | 字符软/硬限制 |
| `summary_blocks` | 2 | 摘要分段数 |

### 1.7 web_search — 网页搜索（Exa）

```json
  "exa": { "api_key": "" }

```

| 字段 | 默认 | 说明 |

### 1.9 ptz — 云台（可选）

```json
{
  "enabled": false,
  "ip": "",
  "port": 80,
  "username": "",
  "password_env": "",
  "step_ms": 300,
  "speed": 0.5
}
```

| 字段 | 默认 | 说明 |
|------|------|------|
| `enabled` | false | 是否启用云台 |
| `ip` | `""` | 摄像头 IP |
| `port` | 80 | ONVIF 端口 |
| `username` | `""` | 用户名 |
| `password_env` | `""` | 密码环境变量名（不填明文） |
| `step_ms` | 300 | 单步转动时长（毫秒） |
| `speed` | 0.5 | 转动速度 0.1~1.0 |

> ONVIF via zeep（同步阻塞），后端用 `asyncio.to_thread` 丢线程池。

### 1.10 home — 家庭信息

```json
{
  "home_name": "童柯诚的家",
  "owner_name": "小童",
  "province": "上海市",
  "city": "上海市",
  "district": "宝山区"
}
```

| 字段 | 说明 |
|------|------|
| `home_name` | 家庭名称 |
| `owner_name` | 主人称呼（前端问候用） |
| `province` / `city` / `district` | 天气定位 |

> **注意**：天气服务实际读的是这里的 `home` 段，不是设置页改的 `user_settings.home_info`。改天气地点要改这里。

### 1.11 weather — 和风天气

```json
  "kid": "KCWHAKR6HC",
  "sub": "2G2D55DC98",
  "private_key": "MC4CAQAwBQYDK2VwBCIE..."

```

| 字段 | 必填 | 说明 |
|------|------|------|
| `host` | 是 | API 主机 |

### 1.12 logging — 日志

### 1.13 external_mcp — 外部 MCP 服务器

```json
[
  { "name": "filesystem", "cmd": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path"] }
]
```

| 字段 | 必填 | 说明 |
|------|------|------|
| `name` | 是 | 服务器标识（工具前缀），白名单 key |
| `cmd` | 是 | 启动命令 |
| `args` | 否 | 命令参数 |

> stdio + JSON-RPC 2.0，连接超时 60s，失败静默跳过。运行时 `/api/mcp/servers` 只能连这里预声明的（白名单防 RCE）。

## 2. .env 环境变量

```ini
# LLM API 密钥（被 llm_keys[].api_key_env 引用）
ZHIPU_API_KEY_2=...
LLM_KEY_API_C12091=...
LLM_KEY_OPEN_F2EF5B=...
LLM_KEY_OPEN_FCC2A5=...

# JWT 鉴权
JWT_SECRET=...                # 不设则自动生成并存 app/data/.jwt_secret

# 可选：API 兜底令牌（JWT 之外的 X-API-Token 认证，留空不启用）
APP_TOKEN=

# 日志级别（覆盖 config.json logging.level）
LOG_LEVEL=INFO

# HA / PTZ / STT 等其它密钥
HA_TOKEN=...
PTZ_PASSWORD=...
SILICONFLOW_STT_KEY=...
```

| 变量 | 说明 |
|------|------|
| `*_API_KEY` / `LLM_KEY_*` | LLM 提供商密钥，被 `api_key_env` 引用 |
| `JWT_SECRET` | JWT 签名密钥；不设则持久化到 `app/data/.jwt_secret`，再没有则每次启动随机（重启失效） |
| `APP_TOKEN` | 兜底令牌，配 `X-API-Token` 头使用 |
| `LOG_LEVEL` | 日志级别 |
| `HA_TOKEN` | HA 访问令牌（也可在 config.json 配） |
| `PTZ_PASSWORD` | 云台密码（被 `ptz.password_env` 引用） |
| `SILICONFLOW_STT_KEY` | 语音转文字密钥 |

> 环境变量优先级高于 `config.json`。LLM Key 的值在 `.env`，名字在 `config.json` 的 `api_key_env`，两层间接。

## 3. docker-compose.yml

只有 **2 个服务**（搜索已改用云端 Exa，无 SearXNG）：

```yaml
services:
  mqtt:
    container_name: mosquitto
    image: eclipse-mosquitto:2
    ports:
      - "1884:1884"
    volumes:
      - ./mosquitto/config:/mosquitto/config
    restart: unless-stopped

  homeassistant:
    container_name: aether-ha
    image: aether-ha:local          # 本地构建，不是 ghcr.io
    ports:
      - "8123:8123"
    volumes:
      - ./ha_config:/config
    restart: unless-stopped
    environment:
      TZ: Asia/Shanghai
```

| 容器名 | 镜像 | 端口 | 挂载 |
|--------|------|------|------|
| `mosquitto` | `eclipse-mosquitto:2` | 1884 | `./mosquitto/config` |
| `aether-ha` | `aether-ha:local` | 8123 | `./ha_config` |

### Mosquitto 配置 (`mosquitto/config/mosquitto.conf`)

```conf
listener 1884
allow_anonymous false
password_file /mosquitto/config/passwd
```

> HA 用本地镜像 `aether-ha:local`（含 MQTT 集成等预装），首次需用配套脚本构建。Mosquitto 关了匿名访问，用 `passwd` 文件做认证。
